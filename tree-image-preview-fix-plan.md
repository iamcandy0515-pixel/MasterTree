# 📋 작업계획서: 수목 비교 및 도감 미리보기 데이터 복구 및 고도화

## 1. 개요 (Overview)
- **목적**: '비교 수목 상세' 및 '수목 도감' 미리보기에서 카테고리(잎, 수피, 꽃, 열매)별 이미지가 누락되는 현상을 완벽히 해결하고, UI 접근성을 개선함.
- **핵심 타겟**: 박달나무(ID: 15), 자작나무 등 이미지 타입 명칭이 불일치하는 모든 수종.

## 2. 문제 분석 (Analysis of Core Issues)
연쇄적 사고를 통해 도출된 데이터 누락의 3대 원인은 다음과 같습니다.
1. **명칭 불일치 (Naming Mismatch)**: DB에는 '잎', 'Leaves', 'leaf' 등으로 혼용되어 있으나, 코드는 'leaf'만 인식함.
2. **UI 로직 누락 (Logic Gap)**: 뷰모델(ViewModel)이 '꽃', '열매' 데이터를 전달하는 분기문이 누락되어 데이터가 있어도 화면에 전달되지 않음.
3. **접근성 저하 (UI/UX)**: '미리보기' 버튼이 본문에 위치하여 사용자가 수목명 확인 후 버튼을 찾기 위해 시선을 옮겨야 하는 불편함이 있음.

## 3. 해결 전략 및 복구 방안 (Recovery Strategy)

### 3.1. [복구] 매핑 테이블(Mapping Table) 기반 정문화
- **전략**: 하드코딩된 `if-else`를 제거하고, 정적 매핑 테이블을 사용하여 모든 변종 명칭을 표준 키로 변환함.
- **적용 대상**: `TreeGroupMember` (비교용), `TreeImage` (도감용)
- **매핑 규칙 (Mapping Table)**:
    - `잎`, `잎새`, `leaves` → **`leaf`**
    - `수피`, `나무껍질`, `bark_skin` → **`bark`**
    - `꽃`, `blossom` → **`flower`**
    - `열매`, `seed` → **`fruit`**

### 3.2. [보완] 뷰모델 및 UI 고도화
- **이미지 폴백(Fallback)**: 카테고리 전용 사진이 없을 경우, 해당 수목의 '대표 사진(imageUrl)'을 기본으로 노출하여 빈 화면 방지.
- **UI 일관성**: 미리보기 탭을 '스마트 태그(ChoiceChip)' 스타일로 변경하여 관리자 앱 전체의 디자인 톤앤매너 유지.
- **버튼 재배치**: 도감 상세 화면의 '미리보기' 버튼을 상단 앱바(AppBar) 수목명 옆으로 이동시켜 접근성 향상.

## 5. 향후 리스크 관리 (Future Risk Management)
- **새로운 별칭 발생 시**: 로직 수정 없이 `_typeMapping` 상수에 새로운 키값만 추가하면 즉시 대응 가능.
- **이미지 로딩 최적화**: 구글 드라이브 이미지 로딩 안정을 위해 프록시 서버 연동 상태 상시 점검(기존 `fix-tree-image-loading.md` 내용 계승).
- **데이터 일관성**: 향후 수목 등록 시 이미지 타입을 선택형(Dropdown)으로 제한하여 데이터 정합성 유지 권장.
