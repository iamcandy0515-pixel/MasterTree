# 인증 시스템 및 세션 관리 안정화 작업 계획서 (Draft)

## 1. 개요
현재 사용자 앱의 인증 시스템에서 발생하는 두 가지 핵심 문제(중복 로그인 허용 및 간헐적인 	oString() Null 오류)를 근본적으로 해결하고, 진정한 의미의 '1인 1기기' 원칙을 실현하기 위한 작업 계획입니다.

## 2. 핵심 이슈 분석 및 해결 전략

### 문제 1: Web 환경의 중복 로그인 차단 미작동
- **원인:** device_info_plus가 웹에서 제공하는 기본 UUID가 브라우저 고유값이 아닌 userAgent를 사용함에 따라, 동일 모델/브라우저 사용자들을 같은 기기로 인식함.
- **해결:** SharedPreferences 또는 브라우저 LocalStorage에 영구적인 고유 UUID를 생성/저장하여 웹 브라우저 단위의 고유 기기 식별 체계 확립.

### 문제 2: 간헐적인 NoSuchMethodError: toString on null
- **원인:** DB 응답 데이터(dynamic) 처리 과정에서 특정 필드가 
ull일 경우 직접적인 .toString() 호출로 인해 발생.
- **해결:** 문자열 보간(${value ?? ''}) 방식 도입 및 BaseApiService의 데이터 캐스팅 로직 강화.

---

## 3. 단계별 추진 계획

### [Phase 1] 기기 식별 체계 강화 (Immediate)
- **대상:** lib/core/device_info_service.dart
- **작업:** 
  - 웹 접속 시 로컬 저장소에 고유 UUID가 있는지 확인.
  - 없을 경우 uuid 패키지 등을 통해 새로 생성하여 영구 저장.
  - 이를 통해 삼성 플립과 Z폴드(혹은 동일 기종의 서로 다른 폰)를 명확히 구분.

### [Phase 2] 실시간 세션 검증 로직 구현
- **대상:** lib/core/api/base_api_service.dart 및 리액티브 감시 로직
- **작업:**
  - 모든 API 요청 시 로컬의 session_id와 DB의 last_session_id 일치 여부 확인 (필요 시).
  - 앱이 포그라운드로 복귀할 때 세션 상태를 재확인하여 다른 기기에서 로그인이 발생했을 경우 즉시 초기 화면으로 튕기도록 처리.

### [Phase 3] 전역 데이터 모델 안정화
- **대상:** lib/utils/quiz_data_mapper.dart, lib/viewmodels/auth_logic_handler.dart 등
- **작업:**
  - 모든 dynamic 맵 데이터 접근 시 	oString() 제거 및 안전한 타입 변환기 적용.
  - 전역 에러 바운더리를 통한 로그 수집 강화.

---

## 4. 기대 효과
- **완벽한 중복 로그인 차단:** 기종이 같더라도 각 브라우저/기기별로 독립적인 세션 관리가 가능해짐.
- **앱 안정성 극대화:** 데이터 누락이나 예기치 않은 NULL 응답으로 인한 앱 크래시 0% 달성.
- **사용자 경험 향상:** "다른 기기에서 사용 중"이라는 안내가 정확한 시점에 제공됨.

## 5. 승인 요청
위 계획에 대해 개발자님의 승인이 떨어지면, 즉시 **Phase 1: 기기 식별 수정**부터 착수하겠습니다.
