# 🧩 고성능 학습 통계 엔진 및 요약 테이블(Summary Table) 설계 작업계획서 (v2.1)

## 1. 개요 (Objective)
기존 학습 통계 산출 방식의 **데이터 부하**와 **논리적 오류(정답률 100% 초과)**를 해결하기 위해, 중복 시도 데이터에서 **최근 시도 결과**만을 관리하는 **요약 테이블(Summary Table)**을 구축하고 통계 엔진을 고도화합니다. 수량 변화(추가/삭제) 시에도 논리적 정합성을 유지하며, 최적화된 마이그레이션 정책을 수립합니다.

## 2. 현상 분석 및 핵심 과제
- **[오류]**: 중복 문제 시도 횟수가 모두 가산되어 정답률이 100%를 초과하는 현상 (예: 1문제를 5번 풀어 5번 맞히면 정답률 500%).
- **[성능]**: 매번 수천 건의 로그를 전수 조사하여 메모리 연산 시 서버/앱 지연 발생.
- **[정규화]**: `quiz_attempts` 내 중복 기록되는 카테고리/세션 정보로 인한 인덱스 부하.
- **[가변성]**: 문항 수 추가/삭제 시에도 지식 습득률(Mastery)의 논리적 모순 방지.

## 3. 세부 설계 및 작업 계획 (To-Do List)

### Phase 1: DB 마이그레이션 및 구조 최적화 (Supabase/PostgreSQL)
- [ ] **불필요한 중복 컬럼 분석 및 인덱싱 정책 수립**
    - `quiz_attempts`에서 통계 산출 시 불필요한 `category_id`, `session_id` 호출 의존성 제거.
    - 원천 로그(`quiz_attempts`)에 `(user_id, question_id, created_at DESC)` 복합 인덱스 추가하여 조회 속도 보장.
- [ ] **전공 요약 테이블 `user_quiz_summary` 신설 (최적화 스키마)**
    - `id`: Primary Key
    - `user_id`: 사용자 고유 ID (FK, Index)
    - `question_id`: 문항 고유 ID (FK - **ON DELETE CASCADE**)
    - `tree_id`: 수목 퀴즈용 (NULL 가능 - **ON DELETE CASCADE**)
    - `exam_id`: 기출 문제용 (NULL 가능)
    - `is_last_correct`: **최근 시도 정답 여부 (Boolean)** - 핵심 지표 (0~100% 보장)
    - `total_attempts`: 누적 시도 횟수
    - `updated_at`: 마지막 학습 시간
- [ ] **초기 데이터 마이그레이션**: 기존 `quiz_attempts` 기록에서 문항별 마지막 결과를 추출하여 `user_quiz_summary`에 UPSERT 작업 수행.

### Phase 2: 서버 서비스 레이어 개편 (Node.js API)
- [ ] **[Rule 1-1]** `stats_user.service.ts` 리팩토링 (메모리 루프 제거 및 SQL 집계 방식 전환).
- [ ] **실시간 동적 분모(Dynamic Denominator) 산출*:
    - 전체 문항 수 실시간 조회(COUNT *)를 통해 수목/문제 추가 시 습득률(%)이 자연스럽게 조정되는 로직 구현.
    - 문항 삭제 시 `CASCADE`를 통해 요약 데이터 동시 정리 (부정 데이터 방지).
- [ ] **성능 기반 집계 API 구현**:
    - `user_quiz_summary`에서 수목 퀴즈와 기출 문제의 전체 수 대비 `mastered_count` 비율 산출.
    - **오답률 감소 추적**: 최근 20개 시도 이력을 별도의 성능 지표(`recent_accuracy`)로 제공하여 변화 확인.

### Phase 3: 사용자 앱 UI/UX 커스텀 (Flutter 3.7.12)
- [ ] **통계 대시보드 탭 분리**: 수목 퀴즈와 기출 문제 탭을 명확히 구분하여 개별 성취도 표시.
- [ ] **직관적 학습 지표 도입**:

    - **학습 성장 배지**: 최근 20개 문항의 정답률 상승 시 시각적 보상 제공.
- [ ] **[Rule 3-2]** `flutter analyze`를 통한 린트 및 UI 레이아웃 점검.

## 4. 기대 효과
1. **정합성**: 데이터 삭제 시 자동 정리가 이루어지며, 정답률이 100% 내로 제어됨.
2. **고성능**: 대량의 로그 데이터를 실시간으로 연산하지 않으므로 데이터 규모에 상관없이 일정한 응답 속도 유지.
3. **유연성**: 새로운 정보(나무/문제)가 추가되더라도 별도의 데이터 초기화 없이 즉각 반영 가능.
