# [제안서] 비교수목 일람 목록 최적화 및 이미지 부하 절감 방안

## 1. 개요
관리자 앱의 '비교수목 일람(Tree Group List)' 화면에서 다수의 이미지를 동시에 로드함에 따라 발생하는 성능 저하와 서버 부하를 해결하기 위한 최적화 전략을 제안함.

## 2. 현재 로딩 부하 분석
- **네트워크 오버헤드**: 리스트 출력 시 모든 그룹의 대표 이미지를 프록시 서버에 요청하며, 평균 50개 이상의 요청이 동시에 발생.
- **서버 CPU 스트레스**: 노드 API 서버가 구글 드라이브 원본을 내려받아 실시간으로 WebP 변환 및 리사이징(Width 80px)을 수행하는 과정에서 연산량이 급증함.
- **UI 반응성 저하**: 이미지 로딩이 완료될 때까지 리스트 스크롤이 불안정하거나 빈 화면이 오래 노출됨.

## 3. 제안: 아이콘 기반 '초고속' 목록 출력 전략
### 3.1 목록 화면 이미지 제거 및 아이콘 대체
- **기존**: `CircleAvatar` 내부에 `CachedNetworkImage`를 사용하여 각 그룹의 실제 사진을 표시.
- **변경**: 실제 이미지 대신 수목을 상징하는 **추상적인 이미지 아이콘(Icons.park, Icons.image_search 등)**으로 대체.
- **효과**: 네트워크 요청 0건, 서버 연산 부하 0%, 목록 렌더링 즉시 완료.

### 3.2 이미지 로드 시점 지연 (Lazy Loading)
- 목록에서는 아이콘만 보여주고, 사용자가 특정 그룹을 클릭하여 **'상세 화면'**으로 진입했을 때만 이미지를 로드함.
- 사용자가 관심을 가지는 특정 데이터에 대해서만 서버 자원을 집중 투자함.

## 4. 작업 계획 (워크플로우)
1. **TreeGroupListItem 분석**: 현재 이미지를 호출하고 있는 `CircleAvatar` 로직 파악.
2. **UI 리디자인**: 아이콘 위젯에 테마 색상(Tree App 전용 그린/다크 톤)을 적용하여 시각적 완성도 유지.
3. **상세 화면 최적화**: 상세 화면 및 미리보기 다이얼로그에서도 썸네일을 우선 로드하는 방식 결합.

## 5. 기대 효과
- **서버 운영 비용 절감**: 프록시 API 호출 횟수가 90% 이상 차단됨.
- **UX 만족도 향상**: 스크롤 시 즉각적인 반응성을 보여주어 "앱이 극도로 빨라졌다"는 체감을 제공함.
- **트래픽 최적화**: 관리자용 웹/모바일 앱의 초기 데이터 전송량 최소화.
