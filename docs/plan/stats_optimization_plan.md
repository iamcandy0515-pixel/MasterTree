## 1. 개요 및 목적
*   **배경**: 현재 `user_quiz_summary` 테이블의 데이터 비대화로 인한 Supabase(500MB) 용량 압박 및 조회 성능 저하 해결 필요.
*   **목표**: 
    1. 통합 로그(`quiz_attempts`)의 **자동 순환(Rotation)** 정책 수립.
    2. **4대 수목 분류** 및 **기출 회차별** 상위 집계 테이블 도입으로 조회 속도 획기적 향상.
    3. 기존 사용자 앱 통계 탭의 **레이아웃(Layout) 고도화**.

## 3. 데이터 아키텍처 설계 (4-Table 모델)

### 3.1 로그 통합 및 자동 순환 (`quiz_attempts`)
*   **정책**: 최신 **100일** 데이터 유지 또는 누적 용량 **200MB** 도달 시 자동 삭제(Purge).
*   **핵심**: 로그 삭제 후에도 상위 집계 데이터는 영구 보존되어 성적 변화 없음.

### 3.2 수목 분류 매칭 (`tree_category_mapping`)
*   **매칭**: DB 원본('침엽수,상록수') ↔ 화면 표시('상록침엽수') 연결.
*   **분류**: 상록침엽수, 낙엽침엽수, 상록활엽수, 낙엽활엽수.

### 3.3 분류/회차별 집계 (`user_tree_category_stats`, `user_exam_session_stats`)
*   **항목**: 전체(Total), 습득완료(Mastered), 도전중(Active), 정답률(Accuracy).
