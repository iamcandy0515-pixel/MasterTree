# 현재 작업 현황

## [1] 작업 계획

- 목표:
    1. 기출문제 PDF 파싱 AI 프롬프트 개선: 계산문제 해설 포맷 구조화 및 수식/계산 오류 자체 검증 강제.
    2. 문제 내용("content_blocks")은 원본을 무조건 완벽하게 유지.
    3. 해설 내용("explanation_blocks")은 계산문제일 경우 4단계 요약 풀이(풀이순서, 공식, 공식대입 및 계산, 정답)를 의무 적용하고, 오답 방지를 위해 분자/분모 대입 등 연산 과정 자체 검증 루틴 지시.
- 범위:
    - `nodejs_admin_api/src/modules/quiz/quiz.service.ts` (`extractQuizFromPdfBuffer`)

## [2] 세부 작업 내용

- [x] `quiz-ai-prompt-update.md` 파일 생성 (완료)
- [x] `quiz.service.ts` 내 프롬프트 수정 반영
    - 계산 문제 전용 포맷 지시어 명확화 (풀이순서 → 공식 → 공식대입 및 계산 → 정답).
    - 분자/분모 오류 방지를 위한 자체 검증 지시 명시.
    - "문제의 질문 텍스트 원본을 어떤 단어도 빼거나 요약하지 말고 있는 그대로 완벽하게 유지하라"는 강제 지시 보강.

## [3] 결과 분석 및 위험 요인

- **결과 (Result):**
    - 백엔드의 PDF 추출 AI 프롬프트를 전면 개편하여, 관리자가 계산문제를 추출할 때 AI가 자체적으로 구조화된 해설(4단계)을 생성하고, 분자/분모 뒤바뀜 같은 흔한 논리 오류를 차단하도록 가이드.
    - 문제 문구 누락 현상을 원천 방지.
- **리스크 분석 (Risk Analysis):**
    - **이론적 한계점:** 아무리 자체 검증 지시를 내려도 LLM(Gemini) 모델의 태생적 한계로 복잡한 계산에서는 미세한 수치 오류(환각)가 생길 수 있습니다. 관리자의 시각적 검증이 최종 방어선입니다.
    - **UX 측면:** 생성된 해설("풀이순서, 공식...")의 텍스트가 조금 길어질 수 있어, 프론트엔드 UI의 `TextField` 영역 스크롤 빈도가 불가피하게 늘어날 수 있습니다.
