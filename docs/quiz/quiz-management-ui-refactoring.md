# 기출문제 일람 화면 UI 개선 및 레이아웃 정리 (quiz-management-ui-refactoring)

## 1. Plan (상태 기록)

**목적 및 범위:**

- `quiz_management_screen.dart`에서 제공되는 '기출문제 일람' 화면의 편의성과 가독성을 높이기 위해 전체적인 UI 리팩토링 진행.
- 기존의 균등 배분(Expanded) 콤보박스를 글자 크기에 반응하는 Text 콤보박스 형태로 변경.
- 불필요하게 낭비되던 카드 내부 정보('검수대기' 배지, 'O년도 N회차' 헤더 텍스트)를 모두 제거하여 심플하게 구성.
- 질문(본문) 텍스트가 너무 길어지는 현상을 방지하기 위해 2줄 이하 말줄임말(...) 처리 유지 및 강화.
- 관리자 권한 필수 기능으로 판단되는 레코드 '삭제' 기능(아이콘)을 각 카드 우측에 추가.
- '신규등록'과 '저장' 버튼을 상단(조회 영역 아래)에 추가하고, 맨 아래 있던 페이지 이동 네비게이션을 화면 중앙의 스크롤 시작부(카드 첫 줄)로 이동시킴.

## 2. Execute (실행)

- `_buildDropdown`: 불필요한 `Container`의 배경색과 테두리(`BoxDecoration`)를 삭제하고 `isExpanded`를 `false` 설정하여 내용에 맞춰 가로폭이 자동 조절되도록 수정.
- 필터 조건 상단 컨테이너: 기존 `Row` + `Expanded` 구조를 `Wrap`으로 대체하여 반응형 폭을 가지게 하고, 위젯 간 자연스러운 줄바꿈이 지원되도록 함.
- 버튼 추가: 필터 조건 랩(Wrap) 바로 아래 우측 정렬로 '신규등록', '저장' `TextButton`을 추가.
- 카드 UI 정리 (`_buildQuizTextCard`): 불필요한 `Column` 및 헤더 정보(과목, 년도, 회차, `_buildStatusBadge`)를 완전히 제거. `Row`를 활용하여 좌측엔 본문 질문(`Expanded`), 우측엔 삭제용 `IconButton`이 위치하는 깔끔한 구조로 수정.
- Pagination 컨트롤 이동: `_buildPagination()` 위젯 호출부를 최하단부에서 데이터(List)가 렌더링 시작되는 윗부분으로 이동 완료.
- 삭제 기능: `_deleteQuiz(int id)` 함수를 비동기로 구현하고, Supabase를 통해 바로 지워지고 `_fetchQuizzes()`를 콜백하여 리스트가 갱신되도록 처리.

## 3. Review (사후 점검)

- **Result (완료된 결과):** 화면이 이전 대비 눈에 띄게 간결해졌으며, 조회 필터 영역은 콤보박스가 텍스트 길이만큼만 자리를 차지하여 시각적 답답함이 해소됨. 네비게이션이 위에 배치되어 리스트 확인과 페이지 이동 시의 동선이 단축됨. 카드 안의 텍스트가 아무리 길어도 레이아웃을 깨지 않고 2줄 말줄임 처리되며, 우측의 아이콘으로 바로 문제를 삭제할 수 있게 됨.
- **Risk Analysis (향후 문제점 및 리스크 분석):** '저장' 및 '신규등록' 텍스트 버튼을 만들었으나 아직 동작 내부 함수가 비어 있는 더미 버튼인 상태임. 추후 타 페이지나 기능과 연결할 때 이벤트 처리를 추가해야 함. 또한, 실수로 삭제하는 것을 방지하기 위해 Alert Dialog를 띄웠으나, 삭제된 데이터를 복원하는 휴지통(Soft Delete) 기능이 없어 DB 측면에서 완전 소멸됨을 관리자에게 인지시킬 필요가 있음.
