# 🛠️ 관리자 시스템 테스트 및 장애 조치 가이드 (Admin Test & Troubleshooting Guide)

관리자 시스템 테스트 시 반드시 확인해야 할 절차와 명령어, 발생 가능한 오류 해결 방법을 정리한 가이드입니다.

---

## 🏗️ 1. 서버 기동 순서 (Priority Order)

서버 간의 의존성 때문에 반드시 **백엔드(API)를 먼저** 기동해야 합니다.

1.  **Step 1: Backend API 서버 기동**
    - 프론트엔드가 실행될 때 API 연결 상태를 체크하므로 백엔드가 준비되어 있어야 합니다.
2.  **Step 2: Frontend Flutter Web 기동**
    - 백엔드가 활성화된 상태에서 웹 브라우저를 띄워 테스트를 시작합니다.

### ⚠️ 서버 연속성 주의사항 (Server Continuity)

- **장시간 미사용 시**: 단순히 화면을 오래 띄워둔다고 해서 서버가 죽지는 않습니다.
- **중단 원인**: 터미널 창(CMD/PowerShell)을 닫거나, 컴퓨터가 **절전(Sleep) 모드**에 들어가면 서버 프로세스가 강제 종료됩니다.
- **복구 시점**: 테스트 도중 화면이 멈추거나 데이터 로드가 안 된다면 터미널의 서버 실행 상태를 가장 먼저 확인하세요.

## 🚨 4. 오류 발생 시 조치 방법 (Troubleshooting)

### ❌ `ERR_CONNECTION_REFUSED` (사이트에 연결할 수 없음)

- **원인**: 서버가 실행 중이 아니거나 비정상 종료됨.
- **해결**: 터미널에서 `npm run dev` 또는 `flutter run` 명령어를 다시 실행하세요.

### ❌ `403 Forbidden` 또는 `401 Unauthorized`

- **원인**: Supabase 세션 만료 또는 권한 부족.
- **해결**: 로그아웃 후 다시 로그인하거나, `.env` 파일의 `SUPABASE_KEY` 설정이 올바른지 확인하세요.

### ❌ `CORS Error` (Network Error)

- **원인**: 보안 정책으로 인해 웹에서 API 호출이 차단됨.
- **해결**: `nodejs_admin_api/src/app.ts`에서 `cors()` 설정에 `http://localhost:8081`이 포함되어 있는지 확인하세요.

### ❌ 이미지 업로드 실패

- **원인**: 파일 크기 초과 또는 Supabase Storage 버킷 설정 문제.
- **해결**: 5MB 이하의 이미지를 사용하고, Supabase 대시보드에서 `tree-images` 버킷의 정책(RLS)이 `Public`인지 확인하세요.

## 💡 팁: 빠른 재시작 (Restart Strategy)

1.  **수동 재시작**: 터미널에서 `Ctrl + C`로 종료 후 "백엔드 -> 프론트엔드" 순으로 다시 시작합니다.
2.  **AI 요청**: 서버가 꺼진 것 같으면 저(Antigravity)에게 **"서버 다시 켜줘"**라고 요청하세요. 제가 즉시 모든 명령어를 실행하여 복구해 드립니다.
