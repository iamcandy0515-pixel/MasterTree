# 📄 기출문제 PDF 추출 설계서 (PDF Extraction Specification)

## 1. 개요

- **목적**: PDF 형식의 기출문제를 AI(Gemini)를 통해 분석하여 정형화된 데이터(JSON)로 추출하는 로직을 정의합니다.
- **주요 대상**: 산림기사, 산림산업기사 등 실기 필답형 PDF 문서.

## 2. 추출 알고리즘 (Extraction Algorithm)

### 2.1 문항 식별 로직

- **검색 트리거**: 문서 내의 `${questionNumber}.`, `Q${questionNumber}`, `문 ${questionNumber}` 패턴을 검색.
- **범위 한정**: 요청된 시작 번호(`start`)부터 끝 번호(`end`)까지의 문항만 정확히 추출.
- **정답 인식**: PDF 원문 내의 **'→'** 기호를 정답 및 해설의 시작점으로 정의.

### 2.2 유형별 정답 및 해설 가공 규칙 (Customized Explanation)

문제의 성격에 따라 AI가 자동으로 판단하여 다음 구조로 데이터를 생성합니다.

#### **[유형 1: 계산 문제] (4단계 구조)**

수치 연산이나 공식 대입이 필요한 문항에 적용됩니다.

1. **풀이순서**: 문제 해결을 위한 접근법 및 핵심 원리 요약.
2. **공식 및 계산식**: 활용되는 수학적/물리적 공식 명시.
3. **공식 및 계산식 적용**: 수치 대입 과정 및 단위 환산 검증 (분자/분모 위치 확인).
4. **정답**: 최종 도출된 계산 결과.

#### **[유형 2: 일반 문제] (3단계 구조)**

이론, 정의, 암기형 문항에 적용됩니다.

1. **핵심개념**: 문제와 관련된 주요 이론 또는 정의 요약.
2. **문제해석**: 지문에서 묻는 의도 및 각 보기의 진위 분석.
3. **풀이 및 정답**: 정답의 명확한 근거와 최종 결과.

### 2.3 시각 정보 해석 (Visual OCR)

- 해설 영역이 그림, 도표, 그래프인 경우 이를 단순 생략하지 않고, 시각적 정보(수치, 관계, 추세)를 텍스트로 읽어내어 논리적인 설명으로 변환합니다.

## 3. 데이터 표준 포맷 (JSON Schema)

```json
{
    "question_number": "번호",
    "content_blocks": "문제 지문 (마크다운 배제)",
    "explanation_blocks": "유형별 맞춤형 해설 (Plain Text)",
    "options": ["정답 1개", "매력적인 오답 1개"],
    "correct_option_index": 0,
    "hint_blocks": ["핵심 힌트 1", "핵심 힌트 2"]
}
```

## 4. 제약 및 규칙

- **Plain Text 준수**: LaTeX, Markdown 문법을 일절 배제하여 가독성 확보.
- **2지 선다 고정**: 학습 효율을 위해 정답 1개와 오답 1개로 구성.
- **환각 방지**: 해당 번호가 PDF에 없을 경우 임의로 지어내지 않고 빈 값 반환.
