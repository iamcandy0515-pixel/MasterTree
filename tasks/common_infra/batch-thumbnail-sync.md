# Task Plan: 수목 이미지 DB 및 구글 드라이브 통합 정비 배치 작업 (Originals & Thumbnails)

구글 드라이브(`TreesQuiz`, `TreesQuizThumbnail`)와 DB 간의 데이터 불일치를 해소하고, 파일명을 표준화하여 이미지 관리 체계를 완전히 정립하는 통합 배치 작업 계획서입니다.

## 1. 개요

- **목적:**
    1. DB에 누락된 원본 이미지 정보(`image_url`)를 드라이브 `TreesQuiz` 폴더에서 찾아 복구.
    2. 기존 썸네일 파일명을 식별 가능하게 변경하고 DB `thumbnail_url` 필드에 동기화.
- **핵심 명명 규칙:** `[수목명]_[카테고리]_thumb.webp`
- **대상:** `trees` 및 `tree_images` 테이블 전체 레코드 (78종 수목 대상)

## 2. 통합 배치 프로세스 (Unified Logic)

### Phase 1: 원본 이미지 복구 (Original Image Reconciliation)

1.  **조건 검색:** `tree_images` 테이블에서 `image_url`이 비어 있거나 누락된 항목을 식별합니다.
2.  **드라이브 폴더 탐색:** `TreesQuiz` 폴더(원본 저장소)를 대상으로 해당 수목명과 카테고리 키워드가 포함된 파일을 검색합니다.
3.  **DB 업데이트:** 유효한 원본 파일을 찾으면 구글 드라이브 URL을 생성하여 `image_url` 필드를 최신화합니다.

### Phase 2: 썸네일 식별화 및 URL 동기화 (Thumbnail Sync)

1.  **기존 썸네일 대조:** `TreesQuizThumbnail` 폴더에서 `fileId`를 기반으로 기존 파일을 찾습니다.
2.  **명명 규칙 적용 & Rename:** 파일명이 난수 등으로 되어 있는 경우, `[수목명]_[카테고리]_thumb.webp` 형식으로 이름을 변경합니다.
3.  **DB 업데이트:** 변경된 파일명이 반영된 새 URL을 `thumbnail_url` 필드에 기록합니다.

### Phase 3: 데이터 무결성 검증 (Cross-Check)

1.  **매칭 결과 확인:** 모든 수목(78종)의 5개 카테고리에 대해 원본과 썸네일이 모두 완비되었는지 체크리스트를 생성합니다.
2.  **로그 기록:** 작업 수행 내역(`batch_reconciliation_log.json`)을 생성하여 필드 업데이트 전/후를 기록합니다.

## 3. 세부 실행 기술 사양

- **Google Drive API:** `files.list` (검색), `files.update` (이름 변경) 기능 활용.
- **Supabase Client:** `id` 기반의 정확한 `UPDATE` 쿼리 수행.
- **예외 처리:** 드라이브에 원본 파일이 아예 없는 경우 리포트 출력 후 패스 (수동 확인용).

## 4. 기대 효과

- **데이터 복구:** DB 누락 데이터를 자동 복구하여 앱 내 이미지 미출력 문제 해결.
- **식별성 향상:** 드라이브 관리자가 별도 툴 없이도 파일명으로 데이터 검증 가능.
- **시스템 안정화:** 원본과 썸네일의 매핑 구조가 명확해짐으로써 추후 유지보수 비용 감소.
