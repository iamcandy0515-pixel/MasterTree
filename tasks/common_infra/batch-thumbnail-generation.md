# Task Plan: 78종 수목 썸네일 일괄 생성 및 표준화 배치 작업

구글 드라이브(`TreesQuizThumbnail`) 폴더 내의 식별 불가능한 썸네일 파일 문제를 해결하고, 전체 78종 수목에 대해 표준화된 명명 규칙을 적용한 WebP 썸네일을 일괄 생성하는 배치 작업 계획서입니다.

## 1. 개요

- **목적:** DB에 등록된 78종 수목의 모든 카테고리별 원본 이미지를 기반으로 표준화된 WebP 썸네일을 자동 생성하고 DB를 업데이트함.
- **대상:** `trees` 테이블에 등록된 78종 수목 및 `tree_images` 테이블의 원본 이미지.
- **핵심 규칙:**
    - **포맷:** WebP (손실 압축, 최적화)
    - **파일 이름:** `[수목명]_[카테고리]_thumb.webp` (예: `소나무_꽃_thumb.webp`)
    - **카테고리 한글 매핑:**
        - `representative` / `main` → `대표`
        - `flower` → `꽃`
        - `bark` → `수피`
        - `leaf` → `잎`
        - `fruit` → `열매`

## 2. 기술 스택 및 라이브러리

- **Language:** Node.js (TypeScript)
- **Database:** Supabase (PostgreSQL)
- **Image Processing:** `sharp` (WebP 변환 및 리사이징)
- **Storage:** Supabase Storage (기존 `TreesQuizThumbnail` 관련 경로 혹은 신규 표준 경로)

## 3. 상세 작업 단계

### Phase 1: 데이터 분석 및 환경 준비

1.  **DB 조회:** 78종 수목 리스트와 각 수목의 카테고리별 원본 이미지 URL(`image_url`)을 전체 로드.
2.  **필터링:** 썸네일이 없거나, 명명 규칙이 표준화되지 않은 항목 식별.

### Phase 2: 배치 스크립트 구현 (`scripts/batch_generate_thumbnails.ts`)

1.  **이미지 다운로드:** `axios`를 사용하여 원본 이미지 URL에서 바이너리 데이터 추출.
2.  **WebP 변환 및 리사이징:**
    - `sharp` 라이브러리를 사용하여 가로/세로 최대 300~400px 수준으로 리사이징.
    - `.webp` 포맷으로 인코딩 (Quality: 80~90).
3.  **파일 명명:** `[TreeName]_[Category]_thumb.webp` 규칙 적용.
4.  **스토리지 업로드:** 생성된 WebP 파일을 스토리지의 지정된 폴더에 업로드.
5.  **DB 업데이트:** `tree_images` 테이블의 `thumbnail_url` 컬럼에 새로 생성된 경로/URL 저장.

### Phase 3: 실행 및 검증

1.  **로컬 테스트:** 1~2종의 수목에 대해 먼저 실행하여 파일 이름과 이미지 품질 확인.
2.  **전체 실행:** 78종 수목에 대해 순차적(또는 병렬 제한) 배치 실행.
3.  **결과 보고:** 작업 성공/실패 리스트 및 생성된 총 파일 수 보고.

## 4. 기대 효과

- **파일 식별 용이성:** 파일 이름만으로 어떤 나무의 어떤 부위인지 즉시 확인 가능.
- **성능 최적화:** WebP 포맷 사용으로 관리자 및 사용자 앱의 로딩 속도 향상.
- **데이터 정합성:** DB와 실제 저장소 파일 간의 연동 강화.

## 5. 리스크 분석 및 대응

- **중복 처리:** 이미 표준화된 파일이 있는 경우 건너뛰는 로직 포함 (재시도 효율성).
- **메모리 관리:** 78종에 대해 대량 작업 시 메모리 누수가 발생하지 않도록 Stream 방식 혹은 순차 처리 방식 채택.
- **대량 업로드 제한:** 스토리지 API의 Rate Limit을 고려하여 적절한 딜레이(Wait) 추가.
