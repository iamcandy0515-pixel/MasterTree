# [복구 계획서] 사용자 로그인 인증 시스템 정상화 및 상태 관리 동기화

## 1. 배경 및 문제 분석

### 1-1. 발생 현황
- **오류 메시지**: "사용기간이 만료되었거나 입장코드가 올바르지 않습니다."
- **증상**: 관리자 앱에서 입장 코드를 정상 설정했음에도 불구하고, 일부 사용자가 로그인을 시도할 때 일괄적인 에러가 발생하며 진입이 불가능함.

### 1-2. 핵심 원인 분석 (연쇄 사고 결과)
1. **입장 코드 기본값 불일치 (Fallback Discordance)**:
    - `nodejs_admin_api` 및 `app_settings` 테이블의 기본 입장 코드는 `1234`로 설정되어 있음.
    - `flutter_user_app`은 서버 통신 실패 시 Fallback 기본값을 `1004`로 사용하고 있어, 네트워크 지연 시 사용자가 입력한 올바른 코드(`1234` 등)가 거부됨.
2. **승인 프로세스 도입에 따른 로직 누락**:
    - 최근 관리자 앱에 '승인 대기(pending)', '승인(approved)', '거절(rejected)' 상태가 도입되었으나, 사용자 앱의 `AuthController`는 이를 세부적으로 구분하지 않고 단순히 `isValidEntryCode` 결과가 `false`이면 통합 에러를 노출함.
3. **만료일 체크 로직 미비**:
    - DB의 `expired_at` 컬럼에 따른 실제 기간 만료 여부를 앱단에서 명확히 판단하여 메시지를 분리하지 않음.

## 2. 복구 작업 계획 (Recovery Steps)

### Phase 1: 인증 핵심 로직 보강 (완료)
- [x] **입장 코드 기본값 동기화**: `SupabaseService.fetchGlobalEntryCode` 및 Fallback 값을 `1234`로 통일.
- [x] **상태별 대응 로직 구현**:
    - `denied`, `expired`: "사용기간이 만료되었습니다. 관리자에게 문의해 주세요."
    - `waiting` (not approved): "승인 대기 중입니다. 관리자 승인 후 이용 가능합니다."
    - `approved`: 정상 로그인 진행.

### Phase 2: 사용자 경험 개선 및 예외 처리
- [ ] **디버깅 로그 강화**: 서버 API 응답값(성공 유무, 코드값)을 `debugPrint`로 상세히 출력하여 향후 장애 대응력 확보.
- [ ] **에러 팝업 세분화**: 단순히 "알림"이 아닌, 상태에 맞는 타이틀(예: '승인 대기', '기간 만료')을 표시하도록 `login_screen.dart` 다이얼로그 수정.

### Phase 3: 최종 검증 (Validation)
- [ ] **관리자 앱 연동 테스트**:
    - 관리자 앱에서 임의 유저를 '거절' 상태로 변경 후 사용자 앱 로그인 시도 -> "기간 만료" 메시지 확인.
    - 입장 코드를 '4321'로 변경 후 사용자 앱 로그인 시도 -> 정상 로그인 확인.
- [ ] **네트워크 장애 시뮬레이션**: API URL을 일부러 틀리게 설정한 후 기본값(`1234`)으로 로그인이 되는지 확인.
