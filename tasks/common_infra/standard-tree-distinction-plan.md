# 기준 78종과 유사 수목(비교 전용) 구분 방안 제안 (standard-tree-distinction-plan)

사용자께서 요청하신 대로, 기존 78종 기준 목록에 없는 '리기테타소나무'와 같은 유사 수목들을 DB에 추가하면서 화면에서는 구별되게 보여주고, 향후 퀴즈에는 출제되지 않게 막는 두 가지 방안을 제안합니다.

---

## 💡 제안 1. 기존 필드를 활용하는 방법 (가장 빠름 / DB 구조 변경 없음)

현재 수목 모델에 있는 `is_auto_quiz_enabled` (bool) 필드와 `category` (String) 필드를 조합하여 사용하는 방식입니다.

1. **퀴즈 제외 처리**:
    - 리기테타소나무 추가 시 `is_auto_quiz_enabled = false` 로 고정하여 저장합니다.
    - 이렇게 하면 나중에 퀴즈 추출 로직에서 `is_auto_quiz_enabled == true` 인 수목만 가져오게 되므로, 자연스럽게 제외됩니다.
2. **화면 푶시 (구별법)**:
    - 등록 시 `category` 파라미터에 "침엽수/상록/유사종" 과 같이 **'유사종'** 키워드를 넣습니다.
    - 또는 뷰모델이나 UI에서 `tree.isAutoQuizEnabled == false` 여부를 감지하여, 수목 카드 이름 옆에 **[비교전용]** 혹은 **[유사종]** 이라는 특별한 뱃지(박스)를 띄워줍니다.

---

## 💡 제안 2. '기준 수목' 전용 DB 필드 추가 (정석적이고 안정적인 방법)

`trees` 테이블에 기준 수목 여부를 명확히 나타내는 필드를 하나 새로 추가하는 방식입니다.

1. **DB 및 모델 업데이트**:
    - DB 테이블에 `is_standard` (boolean) 필드 추가 (기본 78종은 `true`, 리기테타소나무 같은 추가 유사종은 `false` 부여).
    - Flutter의 `Tree` 클래스와 Node.js 서버 모델에 해당 필드 연동.
2. **퀴즈 제외 처리**:
    - 퀴즈 로직에서 항상 `WHERE is_standard = true` 조건만 조회하여 원천적으로 78종만 추출.
3. **화면 표시**:
    - `tree.isStandard == false` 인 경우 그룹 편집이나 수목 리스트 카드에서 눈에 띄는 라벨(**[ 78종 외 ]** 등)을 표시.
