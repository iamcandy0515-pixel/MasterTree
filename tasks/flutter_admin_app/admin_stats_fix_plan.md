# 관리자 앱 통계 데이터 정합성 개선 및 소스 분리 계획서

본 계획서는 `DEVELOPMENT_RULES.md` 표준을 준수하며, 사용자 앱의 통계 수정 사항을 관리자 앱 대시보드 및 상세 통계에 완벽히 반영하기 위한 단계별 절차를 정의합니다.

## 📋 핵심 분석 및 개선 방향
1. **소스 모듈화 (200줄 제한)**: 현재 `stats.controller.ts`가 400줄을 초과하고 있습니다. 이를 기능별로 분리하여 가독성과 유지보수성을 높입니다.
2. **ID 체계 정합성**: `tree_id`(수목 퀴즈)와 `question_id`(기출 문제)를 모두 고려한 글로벌 통계 집계 로직을 구현합니다.
3. **오답 데이터 고도화**: 수목 식별 오류 빈도를 정확히 측정하여 관리자에게 실질적인 인사이트를 제공합니다.
