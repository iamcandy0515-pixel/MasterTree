# [Plan] 기출문제 일괄 추출 UI 복구 및 맞춤형 데이터 추출 (v5)

## 1. 개요 (ANALYSIS)

- **목적**: 관리자 앱의 PDF 일괄 추출 시 진행 상황을 한 화면에 표시하고, 문제 유형(계산형 vs 일반형)에 따라 해설 추출 방식을 최적화하여 복구합니다.
- **주요 요구사항**:
    - **UI 개선**: 1~15번 고정 숫자 박스 (단일 화면 무스크롤), 'PDF 추출' 버튼 클릭 시 그리드 표시.
    - **맞춤형 정답 및 해설 추출**:
        - **[계산 문제] 4단계 구조**: (1) 풀이순서, (2) 공식 및 계산식, (3) 공식 및 계산식 적용, (4) 정답.
        - **[일반 문제] 3단계 구조**: (1) 핵심개념, (2) 문제해석, (3) 풀이 및 정답.
        - **공통 규칙**: 원문 내 '→' 기호를 기반으로 정답 식별, 시각적 정보(그림/표)의 논리적 텍스트 변환 포함.
    - **데이터 구성**: 문제 지문, 2지 선다(정답1+오답1), 힌트 2개.

## 2. 세부 설계 (PLANNING)

### 2.1 UI 구성 방식 (Frontend)

- **그리드 배치**: `BulkExtractionScreen` 내 `Wrap`을 사용한 15개 그리드 배치.
- **가시성 제어**: `isLoading` 상태와 연동하여 'PDF 추출' 버튼 클릭 시 그리드 노출.
- **인터랙션**: 추출 완료된 번호(색상 활성화) 클릭 시 하단 편집 필드 업데이트.

### 2.2 유형별 추출 알고리즘 (Backend / Prompt)

- **유형 판단**: AI가 문항의 성격(수치 계산 여부)을 먼저 판단하도록 지시.
- **계산 문제 프롬프트**: 상세한 공식 대입 및 단위 환산 과정을 포함한 4단계 강제.
- **일반 문제 프롬프트**: 이론적 배경(핵심개념)과 본문 분석(문제해석)을 포함한 3단계 구성.
- **데이터 정제**: LaTeX 및 마크다운을 배제한 순수 평문(Plain Text) 유지.

## 3. 구현 단계 (IMPLEMENTATION)

### Phase 1: 백엔드 프롬프트 고도화 (`quiz-pdf.service.ts`)

- `buildBatchExtractionPrompt` 메서드 내의 해설 가공 규칙을 유형별(계산/일반)로 분기하여 재작성.
- 일반 문제 전용 3단계 구조(핵심개념/문제해석/풀이 및 정답) 정의 추가.

### Phase 2: 관리자 앱 UI 구현 (`BulkExtractionScreen.dart`)

- 15개 고정 그리드 박스 레이아웃 적용 (박스 크기: 26x26, 간격: 4).
- 추출 진행 상황(5건 단위 업데이트)이 그리드에 즉시 반영되도록 `BulkExtractionViewModel` 연동.

### Phase 3: 데이터 품질 검증

- 실제 기출문제를 추출하여 유형별로 의도한 구조(4단계 또는 3단계)가 정확히 생성되는지 전수 검사.

## 4. 사후 점검 및 리스크 분석 (REVIEW)

- **판단 오류**: AI가 계산 문제를 일반 문제로 오인하여 공식 단계를 생략할 리스크 대처 (프롬프트 내 명확한 기준 제시).
- **레이아웃**: 15개가 한 화면에 들어오도록 하되, 모바일 기기 등 작은 해상도에서 오버플로우 발생 여부 점검.
