# [복구 작업계획서] 수목 이미지 추출 상세 화면 고도화 (tree-image-extraction-detail-restoration)

## 0. 화면개요
0-1. **이미지 관리**: 본 화면은 수목의 카테고리별(대표, 수피, 잎, 꽃, 열매) 원본 이미지와 썸네일 이미지를 통합 관리하는 화면입니다.
0-2. **구글 드라이브 연동**: 설정된 구글 드라이브 폴더로부터 원본 및 썸네일 이미지를 가져와 동기화하고, 이미지 파일 선택 및 URL 링크 정보를 관리합니다.
0-3. **썸네일 생성 기능**: 원본 이미지를 활용하여 표준화된 WebP 썸네일을 생성하고 드라이브에 저장하는 기능을 제공합니다.
0-4. **DB 저장**: 최종 확정된 이미지 URL 정보를 데이터베이스에 저장하며, 이미지 변경 여부를 엄격히 대조합니다.

---

## 1. 화면 진입 시 조회 처리 기능
1-1. **DB 데이터 로드**: `tree_images` 테이블에서 해당 수목의 부위별 '원본 이미지 URL' 및 '썸네일 이미지 URL' 정보를 가져옵니다.
1-2. **설정 정보 참조**: '설정' 메뉴에 등록된 '원본 이미지 구글 드라이브 폴더 ID'와 '썸네일 구글 드라이브 폴더 ID' 정보를 로드합니다.
1-3. **동기화 출력**: 저장된 폴더 정보를 사용하여 드라이브 내 실제 파일 존재 여부를 확인하고, 관련 이미지와 URL 정보를 화면에 출력합니다.
1-4. **실재 확인 및 예외 처리**: DB에 등록된 URL 정보가 실제 드라이브에 존재하지 않을 경우 '없음' 혹은 '삭제됨' 표시를 시각적으로 노출합니다.
1-5. **배지 표시 (DB)**: DB에서 직접 가져온 정보인 경우, 해당 이미지 박스 우측 상단에 **'DB 정보'** 문구를 출력합니다.

## 2. '썸네일 생성' 기능
2-1. **경로 참조**: '설정'에 저장된 원본 폴더 및 썸네일 전용 폴더 정보를 가져옵니다.
2-2. **원본 소스 확보**: DB 혹은 현재 화면에 스테이징된 원본 이미지 URL 정보를 확인합니다.
2-3. **드라이브 통신**: 확인된 URL 정보를 기반으로 구글 드라이브 원본 폴더에서 해당 이미지 데이터를 가져옵니다.
2-4. **OAuth2 인코딩**: **OAuth2 Refresh Token** 방식을 적용하여 서버 사이드에서 안전하게 이미지를 처리하고 썸네일을 생성합니다.
2-5. **명명 규칙 적용**: 생성 시 `[수목명]_[카테고리]_thumb.webp` (예: 소나무_잎_thumb.webp) 규칙을 엄격히 적용합니다.
2-6. **드라이브 저장**: 생성된 WebP 썸네일을 구글 드라이브의 '썸네일 폴더'에 업로드합니다.
2-7. **DB 연동**: 성공적으로 저장된 썸네일의 신규 URL을 DB의 `thumbnail_url` 필드에 즉시 업데이트하거나 저장 준비 상태로 만듭니다.
2-8. **UI 동기화**: 화면의 썸네일 이미지 Box와 'URL 설정' 필드에 생성된 정보를 실시간으로 연결합니다.
2-9. **배지 표시 (Google)**: 구글 드라이브에서 실시간으로 생성하거나 가져온 정보일 경우, 썸네일 이미지 우측 상단에 **'구글 정보'** 문구를 출력합니다.

## 3. DB 저장 기능
3-1. **사전 검증 (Check)**: 저장 전 원본/썸네일 URL의 유효성(null 여부 등)을 체크하고 에러 처리를 수행합니다.
3-2. **선택적 업데이트**: 저장 시 신규 정보가 있는 경우에만 반영하며, 입력값이 없는(null 포함) 카테고리는 기존 DB 정보를 유지하도록 처리합니다.
3-3. **중복 데이터 방지**: DB의 현재 URL 정보와 화면의 수정된 URL 정보가 동일한 경우, **'url 정보가 동일하여 db저장을 할수 없다, 확인하세요'**라는 플로팅 메시지를 띄우고 실제 저장은 수행하지 않습니다.

## 4. 구글 드라이브 연동 기능
4-1. **폴더 매핑**: '설정'의 원본 및 썸네일 폴더 정보를 호출합니다.
4-2. **패턴 검색**: 해당 수목 이름이 포함된 5대 카테고리(대표, 수피, 잎, 꽃, 열매) 원본/썸네일 이미지 및 URL 정보를 드라이브 API를 통해 검색합니다.
4-3. **화면 매치**: 검색된 드라이브 정보를 카테고리별 슬롯에 자동으로 배치하고 매칭합니다.
4-4. **배지 표시 (Google)**: 구글 드라이브 연동을 통해 가져온 정보인 경우, 이미지 박스 우측 상단에 **'구글 정보'** 문구를 출력합니다.
