# Task: Quiz Extraction UI Refactoring (Hint UI & Final Polish)

## 1. ANALYSIS (연구 및 분석)

- **힌트 설정 UI 개선**: 현재는 '힌트 1', '힌트 2'와 같이 라벨이 구분되어 개별적으로 보임.
- **요구사항**: '힌트'라는 공통 제목 아래에 2개의 입력창을 나열하여 시각적으로 하나의 그룹처럼 보이게 함.
- **공통 변경 사항 반영**: 이전 단계에서 구현한 '문제/해설 인라인 이미지' 및 '유사 문제 추천 UI'와 조화를 이루도록 함.

## 2. PLANNING (작업 단계별 계획)

### 1단계: 힌트 모듈 UI 개편

- `5_hint_module.dart` 수정:
    - 타이틀을 '힌트'로 변경 (기존 '힌트 설정').
    - '힌트 1', '힌트 2' 라벨(labelText)을 제거하고, 힌트 입력창들을 공통 컨테이너에 배치.
    - 힌트 개수 선택 드롭다운은 유지하되, 전체적인 시각적 통일성 확보.

### 2단계: 이미지 인라인 표시 고도화 (검토)

- `3_question_explanation_module.dart`:
    - 인라인 이미지가 표시될 때 스크롤이 자동으로 이미지 위치를 추적하도록 개선할 지 검토.
    - 이미지 로딩 속도 및 오류 처리에 대한 최종 점검.

### 3단계: 유사 문제 추천 UI 최종 수정

- `6_related_question_module.dart`:
    - 사용자 앱의 디자인을 완벽히 이식했는지 다시 한 번 대조 및 조정.
    - AI 분석 결과가 없을 때의 Empty State 디자인 강화.

### 4단계: 최종 통합 테스트

- 모든 모듈이 한 화면에서 조화롭게 작동하는지 확인.
- `flutter clean` 및 `build`를 통한 배포 환경 검증.

## 3. SOLUTIONING (아키텍처/디자인 제안)

- **Hint Section Layout**:
    ```
    [힌트] (AI 힌트 생성 버튼)
    -----------------------
    [ 입력창 1 ]
    [ 입력창 2 ]
    ```
    형태로 구성하여 사용자가 '힌트'라는 하나의 세트임을 직관적으로 이해하도록 함.

## 4. IMPLEMENTATION (구현 계획)

- [ ] `5_hint_module.dart` UI 수정 (라벨 제거 및 그룹화).
- [ ] 전체 모듈 레이아웃 간격(Padding/Margin) 최종 조정.

## 5. RISK ANALYSIS (리스크 분석 및 사후 점검)

- **라벨 제거 효과**: 라벨이 없으면 각 입력창의 용도가 불분명해질 수 있으므로, `hintText`를 활용하여 '첫 번째 힌트', '두 번째 힌트' 등으로 내용을 안내.
