# 일괄 유사문제 추출 UI 개선 작업 계획 (Bulk Similar Management UI Plan)

## 📌 목표 (Plan)

`BulkSimilarManagementScreen` 내비게이션 및 핵심 동작 버튼의 배치를 개선하고 버튼 종류를 통일(TextButton)하여 화면을 보다 넓고 시원하게 사용할 수 있도록 UI를 개편합니다.

## 🛠 작업 범위 및 세부 변경 사항

### 1. 버튼 텍스트 변경

- **변경 전**: `일괄 유사문제 적용 (Batch 5)`
- **변경 후**: `일괄 유사문제 추출 (page단위)`

### 2. 버튼 배치 및 스타일 변경 (상단 이동)

- **기존**: 리스트 최하단(`_buildActionButtons`)에 꽉 찬 `OutlinedButton`과 `ElevatedButton` 형태의 버튼 바로 고정 배치됨.
- **변경**: 하단 고정 바 구조(`_buildActionButtons`)인 `Expanded` 및 `Container` 영역 자체를 제거. 해당 두 가지 버튼(`일괄 유사문제 추출(page단위)`, `일괄 저장`)을 모두 **`TextButton`** 타입으로 변환하고, 화면 최상단 제목 바로 아래 영역(또는 `AppBar`의 `actions` 버튼 집합, 혹은 `_buildFilterArea` 직전/상단부)으로 깔끔하게 이동하여 접근성을 높입니다.

### 3. '조회' 버튼 스타일 변경

- **대상**: 필터 영역(`_buildFilterArea`) 우측 끝에 위치한 **조회** 버튼
- **기존**: 파란색의 꽉 찬 `ElevatedButton`
- **변경**: 이를 심플한 `TextButton`으로 변경하여 검색(필터) 바의 전반적인 디자인 톤 앤 매너(T&M)를 미니멀하게 통일합니다.

## ⚠️ 리스크 분석 (Risk Analysis)

- **UI 여백 및 터치 영역 이슈**: 하단에 크게 렌더링되던 버튼이 텍스트 형태의 위쪽 버튼으로 이동하게 되므로, 실수로 터치하지 않도록 버튼 간 충분한 패딩(Margin/Padding)을 확보해야 합니다.
- **버튼 비활성화 피드백 (State)**: 데이터가 비어있거나 API 처리 중(`_isProcessing`)일 때 `TextButton` 색상이 완전히 투명해져서 못 누르는 것처럼 보일 수 있는 부분에 대해 기존처럼 시각적인 비활성 처리(회색톤 등)가 온전히 넘어가도록 속성을 명시적으로 할당해주어야 합니다.

이 작업 계획으로 진행해도 괜찮을지 승인해주시면, 실제로 파일에 수정 코드를 즉시 적용하겠습니다!
