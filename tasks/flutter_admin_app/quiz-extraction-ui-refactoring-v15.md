# Task: Quiz Extraction - Related Questions UI Minimalist & Space Optimization (v15)

## 1. ANALYSIS (연구 및 분석)

### 1-1. 요구사항 정밀 분석

- **카드 스타일 제거**: 메인 화면의 '유사 기출문제' 리스트 항목에서 **배경색(BoxDecoration)과 테두리(Border)를 완전히 삭제**하여 투명한 리스트 형태로 변경.
- **수직 공간 극대화**: 항목 간의 여백(Vertical Spacing) 및 내부 패딩을 최소화하여 한 화면에 더 많은 데이터가 노출되도록 조정.
- **가로 공간 재배치**:
    - 정보 배지(과목, 년도, 회차, 번호) 영역의 너비를 **Overflow가 발생하지 않는 최소한의 너비**로 축소하여 왼쪽으로 밀착.
    - 확보된 가로 여유 공간을 **문제 지문 영역(Expanded)**에 할당하여 더 많은 내용을 한눈에 확인 가능하게 함.

## 2. PLANNING (작업 단계별 계획)

### 1단계: 카드 레이아웃 슬림화 (`6_related_question_module.dart`)

- `_buildRelatedQuizCard`의 `Container`에서 `decoration` 및 `border` 삭제.
- 전체 아이템의 `Padding`을 `bottom: 8.0`에서 `4.0`으로 축소.
- 가로 `Row` 내의 정보 배지 너비를 `140`에서 `115` 내외로 최적화 (텍스트 크기 10pt 기준 안전 범위).

### 2단계: 텍스트 밀도 및 가독성 조정

- 원본 텍스트에 이미 `ellipsis`가 적용되어 있으므로, 가로 폭 증가에 따른 가시 필드 확대 확인.
- 정보 배지의 배경색은 옅게 유지하여 문제 번호 간의 구분선 역할은 수행하되 시각적 부담은 감소.

### 3단계: 모달 UI 동기화 (선택 사항)

- 메인 화면의 변경 사항을 참고하여 모달 내의 간격도 유사하게 조정하여 통일감 부여.

## 3. SOLUTIONING (아키텍처/디자인 제안)

- **High-Density Data Grid**: 관리 기능을 수행하는 화면 특성에 맞춰 장식적 요소보다는 데이터 밀도를 우선순위에 둔 설계.
- **Flexible Content Allocation**: 고정 너비 최소화와 가변 영역(지문) 최대화를 통해 다양한 화면 크기에서도 정보 전달력 극대화.

## 5. RISK ANALYSIS (리스크 분석 및 사후 점검)

- **Overflow 리스크**: 배지 너비를 너무 줄이면 '산림산업기사 2024 1회 10' 등의 긴 텍스트가 잘릴 수 있음 -> `115~120px` 정도로 테스트하여 안전 영역 확보.
- **경계 모호성**: 테두리가 사라지므로 항목 간의 심리적 구분감이 약해질 수 있음 -> 보라색 배지의 존재감이 이를 보완하도록 함.
