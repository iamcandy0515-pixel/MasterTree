# 관리자 앱 통계 서비스 복구 및 사용자 중심 개편 작업 계획서 (V6)

## 1. 개요
관리자 앱의 통계 기능을 사용자 활동 중심으로 재설계하고, 상세 통계 화면을 사용자 앱과 동일한 로직 및 UI로 통일합니다. 데이터의 시의성과 운영 효율성을 높이는 것을 목표로 합니다.

## 2. 주요 변경 사항 및 요구사항

### 2.1 통계 메인 화면 (사용자 목록)
- **탭 구분**: '활동 중인 유저'와 '비활동 사용자' 두 개의 탭으로 나누어 표시.
- **필터링**: 상태(`status`)가 'rejected' (거절된 사용자)인 유저는 목록에서 제외.
- **정렬**: 최근 통계 데이터(퀴즈 풀이 등)가 발생한 사용자부터 우선순위로 정렬.
- **표시 정보**: [사] 이름 (수목: N건, 기출: M건) 형태의 텍스트 리스트 유지.

### 2.2 개인별 상세 통계 화면
- **로직 통일**: 사용자 앱의 '나의 학습 통계'(`user_stats_screen.dart`)와 100% 동일한 계산 로직 및 레이아웃 적용.
- **디자인 유지**: 3단 탭(종합, 수목퀴즈, 기출문제) 구조 및 진행률 바(LinearProgressIndicator) 중심의 UI 유지.
- **그래프 배제**: 진행률 이외의 성적 추이 그래프는 절대로 추가하지 않음.

## 3. 상세 작업 단계

### Step 1: 백엔드 API 고도화 (nodejs_admin_api)
- **파일**: `stats.controller.ts`
- **수정 내용**:
    - `getAdminDetailedStats` 함수 내에서 `rejected` 유저 필터링 로직 추가.
    - 각 유저의 최신 활동(퀴즈 시도 내역 중 가장 최근 시간)을 기반으로 정렬.
    - 활동 유저(최근 30일 내 활동 있음)와 비활동 유저를 구분하여 반환하거나, 프론트엔드에서 구분할 수 있는 플래그 추가.

### Step 2: 통계 메인 화면 개편 (flutter_admin_app)
- **파일**: `statistics_screen.dart`, `statistics_viewmodel.dart`
- **수정 내용**:
    - `DefaultTabController`를 사용하여 '활동 중인 유저' / '비활동 사용자' 탭 구현.
    - ViewModel에서 필터링된 유저 리스트를 상태별로 관리.

### Step 3: 상세 통계 화면 로직 이식 (flutter_admin_app)
- **파일**: `user_detail_stats_screen.dart`, `user_detail_tabs.dart`
- **수정 내용**:
    - 사용자 앱의 `UserStatsScreen` UI를 관리자 앱 테마에 맞춰 이식.
    - `UserDetailViewModel`이 특정 `userId`에 대한 상세 통계를 안정적으로 호출하도록 구현.
    - `FittedBox` 등을 사용하여 텍스트 오버플로우 방지.

## 4. 기술 규성 및 안정성 확보 (DEVELOPMENT_RULES.md 준수)

### 4.1 코드 품질 관리
- **200줄 제한**: 각 파일은 200줄 내외로 유지하며, 초과 시 위젯/로직을 레이어별로 분리.
- **Linter 체크**: 작업 완료 후 `flutter analyze`를 수행하여 모든 경고 제거.
- **강력한 예외 처리**: 데이터 로딩 실패 시 '다시 시도' 버튼 및 에러 뷰 제공.

### 4.2 환경 및 보안
- **Windows 11 대응**: 모든 터미널 명령 시 `chcp 65001`을 사용하여 한글 출력 보장.
- **데이터 무결성**: Supabase의 최신 활동 데이터를 실시간으로 반영.
- **승인 프로세스**: 본 계획서 승인 전까지 실제 구현 코드는 엄격히 통제.

## 5. 작업 결과 검증 계획
1. 활동 유저 탭에 최근 퀴즈를 푼 유저가 최상단에 나오는지 확인.
2. 거절된 사용자(`rejected`)가 목록에 노출되지 않는지 확인.
3. 상세 화면이 사용자 앱의 통계와 동일한 데이터(정답률, 진행도 등)를 보여주는지 대조.
4. 모든 프론트엔드 파일의 라인 수 확인.
