# Task: Quiz Extraction - Similar Question Modal & Multi-Stage Sync Refactoring

## 1. ANALYSIS (연구 및 분석)

### 1-1. 요구사항 분석

- **UI 흐름**:
    1. '유사문제 추출' 클릭 -> AI 분석 수행.
    2. 분석 결과 건수 표시 + **[보기]** 버튼 노출.
    3. **[보기]** 클릭 시 모달(Dialog) 창에서 유사 문제 편집(삭제).
    4. **모달 [닫기] 클릭 시**: 편집된 유사 문제 리스트를 메인 화면 '유사 기출문제' 섹션 하단에 즉시 렌더링.
- **DB 저장 로직**:
    - 메인 화면의 'DB 저장' 버튼 클릭 시, 모달에서 최종 확정된 유사 문제 리스트를 함께 저장.
    - **Upsert 전략**: 기존 레코드가 있으면 `related_quiz_ids`를 포함한 전체 데이터를 업데이트하고, 없으면 신규 저장하여 데이터 무결성 보장.

### 1-2. 기술적 구현 방안

- **상태 관리**: `QuizExtractionStep2ViewModel`의 `relatedQuizzes` 리스트를 모달과 메인 화면이 공유.
- **모달 연동**: `StatefulBuilder` 또는 별도 다이얼로그 위젯을 통해 실시간 편집 반영.
- **UI 구성**: `6_related_question_module.dart`를 수정하여 버튼 영역과 리스트 노출 영역을 조건부로 렌더링.

## 2. PLANNING (작업 단계별 계획)

### 1단계: UI 명칭 및 버튼 영역 개편

- 'AI 유사 기출문제 분석' -> **'유사문제 추출'** 라벨 변경.
- 추출 전/후 상태에 따라 '추출된 유사 문제: N개' 메시지와 **[보기]** 버튼 배치.

### 2단계: 편집 모달(Dialog) 구현

- `BulkSimilarManagementScreen` 스타일의 AlertDialog 구현.
- 각 문제 카드 우측에 'X' 아이콘 배치하여 리스트에서 즉시 제거 기능 연결.

### 3단계: 모달 닫기 후 메인 화면 리스트 노출 로직

- 모달이 닫히는 시점에 상태를 동기화하여, 메인 화면의 '유사 기출문제' 타이틀 아래에 편집 완료된 리스트 카드를 세로로 나열.

### 4단계: 통합 Upsert 저장 로직 검토

- `7_db_registration_module.dart`에서 저장 시 `vm.relatedQuizzes`의 최신 ID 목록을 페이로드에 포함하는지 재확인.
- 백엔드에서 `related_quiz_ids` 필드가 정상적으로 업데이트(Upsert)되는지 검증.

## 3. SOLUTIONING (아키텍처/디자인 제안)

- **Interactive Sync**: 모달에서의 삭제 행위가 ViewModel의 상태를 직접 조작하게 하여 별도의 '적용' 버튼 없이 '닫기'만으로도 메인 화면과 동기화되도록 설계.
- **Resource Cleanup**: 이미 구현된 이미지 자동 삭제 로직과 결합하여, 유사 문제 ID 관리도 원자적(Atomic)으로 처리.

- [ ] `_showSimilarQuizzesModal`: 관리용 다이얼로그 위젯 구현.
- [ ] 메인 화면 리스트 카드 렌더링 영역 (모달 닫기 후 노출용) 수정.

## 5. RISK ANALYSIS (리스크 분석 및 사후 점검)

- **UX 혼선**: 모달에서 삭제한 내용이 메인 화면에 즉시 반영되므로, 관리자가 실수로 삭제했을 경우를 대비해 '추천 다시 받기(유사문제 추출)' 기능을 통해 초기화 가능함을 확인.
- **데이터 일관성**: 저장되지 않은 상태에서 화면을 이탈할 경우 데이터가 유실될 수 있으므로, 변경 사항이 있을 때만 저장 버튼을 강조하는 UI 고려 가능.
