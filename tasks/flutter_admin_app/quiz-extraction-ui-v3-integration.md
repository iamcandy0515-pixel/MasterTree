# Task: 캡처 기반 통합 헤더 구현 및 5건 단위 청크(Chunk) 방식 배치 추출 시스템 구축 (v5)

## 1. 개요 (Overview)

관리자 앱의 '퀴즈 상세 추출 및 검토' 화면의 상단 UI를 캡처 화면 사양으로 전면 개편합니다. 또한, 시스템 부하 분산과 사용자 경험 최적화를 위해 '일괄 추출' 화면의 **5건 단위 청크(Chunk) 기반 배치 추출 로직**을 이식하여, 지정된 범위(1~15)의 문제를 효율적으로 추출하고 상세 검토 환경을 완성합니다.

## 2. 작업 범위 (Scope)

- **통합 추출 헤더 (캡처 화면 사양)**:
    - **1행**: 파일명 박스 (녹색 문서 아이콘 포함).
    - **2행**: [과목 | 년도 | 회차] 가로 통합 필터.
    - **3행**: '범위:' 라벨 + From/To 드롭다운(1~15) + **[✨ PDF 추출]** 버튼.
- **부하 분산형 추출 로직 (Chunking)**:
    - `PDF 추출` 버튼 클릭 시, 선택한 범위를 **5문항 단위**로 나누어 서버에 순차적/비동기 요청.
    - 추출된 데이터를 `Map<int, Map<String, dynamic>>`에 즉시 저장하여 메모리에서 즉각 전환 가능하도록 함.
- **워크플로우 고도화**:
    - 추출 진행 상황을 화면 중앙 또는 SnackBar로 시각화 (예: "5/15 완료").
    - 번호 이동 시 메모리 데이터를 컨트롤러로 로드.
    - `DB 저장` 완료 시 다음 번호 자동 이동 및 데이터 자동 로드.
- **UI 최적화**: 섹션 헤더 및 개별 '조회' 버튼 삭제.

## 3. 상세 단계 (Plan)

### Phase 1: ViewModel 부하 분산 로직 구현 (`QuizExtractionStep2ViewModel`)

- [ ] `_extractedQuizzes` 상태 맵 추가.
- [ ] `startBatchExtractionAction` 메서드 구현:
    - 범위를 5개 단위 리스트(`List<List<int>>`)로 분할(Chunking).
    - 각 청크별로 `extractBatch` API 호출 및 진행률 업데이트.
- [ ] `updateEditorFields(int qNum)`: 선택된 번호의 데이터를 UI 텍스트 컨트롤러와 동기화.

### Phase 2: 캡처 기반 통합 헤더 위젯 구현

- [ ] `UnifiedExtractionHeader` 위젯 제작 (또는 기존 모듈 대폭 리팩토링):
    - 캡처 화면의 색상, 간격, 3행 구조 적용.
    - '범위:' 섹션에 시작/끝 드롭다운 배치.
    - **[PDF 추출]** 버튼에 청크 기반 추출 시스템 연결.

### Phase 3: Screen 레이아웃 통합 및 자동화 로직 연결

- [ ] `QuizExtractionStep2Screen`:
    - 상단 기존 모듈들을 단일 헤더로 교체.
    - `DB 저장` 성공 시: 현재 화면 데이터를 맵에 업데이트 -> DB 반영 -> 다음 번호(End 이내) 이동 -> 데이터 로드 자동화.
