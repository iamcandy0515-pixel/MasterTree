# 관리자 대시보드 통계 복구 및 네비게이션 정돈 (admin-dashboard-recovery)

## 1. 상태 기록 (Plan)

- **목적**:
    1. 비어있는 관리자 통계 기능을 사용자 앱과 동일한 API를 사용하여 복구.
    2. 상단 통계 바에서 '유저 정보'를 삭제하여 사용자 앱과 완전히 동일한 3개 항목(수목, 기출, 유사)으로 구성.
    3. 하단 네비게이션 아이템(홈, 통계, 사용자, 설정)을 최신 작업 표준에 맞춰 기능을 유지하며 UI/UX 개선.
- **작업 범위**:
    - `flutter_admin_app/lib/features/trees/repositories/tree_repository.dart` (API 연결 구현)
    - `flutter_admin_app/lib/features/dashboard/screens/dashboard_screen.dart` (통계 바 수정 및 하단 네비게이션 정돈)

## 2. 실행 (Execute)

- [x] **통계 API 연결**: `TreeRepository.getDashboardStats()`를 구현하여 서버(`nodejs_admin_api`)로부터 실시간 통계 데이터를 가져오도록 수정 완료.
- [x] **상단 통계 바 수정**: `_buildTextStatsSection`에서 '유저' 항목 및 구분선을 제거하고 `수목`, `기출`, `유사` 3개 항목으로 재배치 완료.
- [x] **하단 네비게이션 정돈**:
    - '홈', '통계', '사용자', '설정' 네 개의 아이콘과 라벨이 명확하게 보이도록 UI 개선 완료.
    - 각 아이템 클릭 시 해당 스크린(`StatisticsScreen`, `UserCheckScreen`, `SettingsScreen`)으로 올바르게 네비게이션 연결 완료.
- [x] **UI 일관성 체크**: 사용자 앱의 대시보드와 통계 바의 폰트, 간격, 스타일이 일치하는지 최종 확인 완료.

## 3. 사후 점검 (Review)

- **완료된 결과(Result)**:
    - 관리자 대시보드에 실시간 통계 데이터(수목, 기출, 유사)가 사용자 앱과 동일한 방식으로 복구되었습니다.
    - 상단 지표에서 '유저' 정보를 제거하여 관리 항목을 최적화했습니다.
    - 하단 네비게이션을 홈, 통계, 사용자, 설정으로 재구성하여 편의성을 강화했습니다.
- **향후 문제점 및 리스크 분석(Risk Analysis)**:
    - 하단 네비게이션에 '사용자' 항목이 추가됨에 따라 기존 상단 '바로가기' 영역에 있던 사용자 버튼과의 중복 여부를 검토하여 레이아웃을 최적화해야 함.
    - API 통계 데이터 로드 시 네트워크 지연이 발생할 경우를 대비해 초기값(0) 처리 및 스켈레톤 UI 적용 고려.
