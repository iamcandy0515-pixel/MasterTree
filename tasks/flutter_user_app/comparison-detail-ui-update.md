# Task: 유사종 비교 상세 화면 UI 개편

## 1. 상태 기록 (Plan)

- **목적**: `SpeciesComparisonDetailScreen`의 헤더 구성을 변경하고 불필요한 액션 버튼을 제거하며, '잎'과 '수피' 태그를 이용한 실제 이미지 비교 섹션을 강화함.
- **작업 범위**:
    1.  **헤더 수정**: 중앙의 '유사종 비교' 텍스트를 제거하고 그 자리에 '잎', '수피' 텍스트 태그(Toggle 형태) 배치.
    2.  **공유 및 이동 버튼 제거**: 오른쪽의 공유 아이콘과 하단의 '다음 가기' 버튼 삭제.
    3.  **이미지 연동**: 상단 이미지 영역을 선택된 태그('잎' 또는 '수피')에 맞는 실제 수목 이미지로 보여주도록 로직 수정.
    4.  **상태 관리**: `StatelessWidget`에서 `StatefulWidget`으로 변경하여 현재 선택된 태그('잎'/'수피') 상태 관리.

---

## 3. 사후 점검 (Review)

- **완료된 결과(Result)**:
    - **헤더 태그 시스템**: 기존의 '유사종 비교' 텍스트를 제거하고 '잎', '수피' 태그를 중앙에 배치하여 사용자가 비교 대상을 직관적으로 전환할 수 있게 했습니다.
    - **이미지 다이내믹 렌더링**: 선택된 태그(`_selectedTag`)에 따라 좌우 이미지가 해당 수종의 부위별 이미지로 즉시 교체되도록 구현했습니다.
    - **UI 간소화**: 불필요한 '공유하기' 아이콘과 '다음 가기' 버튼을 제거하여 비교 정보에만 집중할 수 있는 깔끔한 레이아웃을 완성했습니다.
    - **가독성 개선**: 텍스트 가이드 메시지도 선택된 태그에 맞춰 유동적으로 변경되도록 처리했습니다.

- **향후 문제점 및 리스크 분석(Risk Analysis)**:
    - **이미지 로딩 속도**: 태그 전환 시 이미지를 새로 불러오므로 네트워크 상태에 따라 지연이 발생할 수 있습니다.
    - **이미지 자산 확보**: 각 수종별로 '잎'과 '수피'에 해당하는 정확한 이미지 데이터베이스가 구축되어 있어야 합니다.
