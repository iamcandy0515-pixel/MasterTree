# 기출문제 연동 화면(PDF 모듈) Dropdown 렌더링 및 오버플로우 오류 수정

## 1. 목적 및 원인 파악 (Plan)

- **증상:** 사용자가 `PDF 문제추출` 탭이 있는 '기출문제 연동' 화면에 진입하거나, 구글 드라이브 파일 검색을 통해 특정 PDF(`산림필답...` 등)를 로드하는 순간 크래시(빨간 화면)와 함께 `RenderFlex overflow` 발생.
- **원인 (Assertion failed):**
    - `DropdownButton` 컴포넌트(`_buildDropdown`)는 매개변수로 전달받은 `value`가 하위 `items` 목록에 **반드시 1개 존재**해야만 렌더링에 성공합니다.
    - 하지만 드라이브의 파일 이름에서 추출해 낸 `과목(extractedSubject)` 값이 `"산림필답"`으로, 우리가 하드코딩해둔 기본 6가지 항목(`['산림기사', '산림산업기사', '산업안전기사', '산업안전산업기사', '조경기사', '조경산업기사']`) 목록에 없었기 때문에 Flutter 내부 치명적 예외(Assertion Error)가 발생했습니다.
- **원인 (RenderFlex overflow):**
    - Dropdown이 터지면서 에러를 출력하는 `ErrorWidget(빨간 바탕에 노란 글씨)`이 생성되었고, 이 위젯은 화면 가로 폭을 무시하고 긴 에러 메시지를 가로로 쭉 출력하기에 `Wrap`을 뚫고 나가면서 18px, 105px, 81px 등의 연쇄적인 **오버플로우 UI 깨짐 현상**을 발생시켰습니다.

## 2. 작업 내용 (Execute)

- `2_pdf_extraction_module.dart`:
    - **선택된 값 동적 추가 방어 (Fallback List Injection):** `vm.extractedSubject`, `vm.extractedYear`, `vm.extractedRound` 값이 존재하지만 하드코딩된 기본 드롭다운 `items` 배열(과목/연도/회차)에 없다면, 해당 값을 즉석에서 배열에 `add()`하여 목록에 포함시키도록 안전 조치를 취했습니다.
    - **문제 번호(Q) 범위 확장 조치:** 기본적으로 20번까지만 드롭다운에 표시되던 부분을, `vm.selectedQuestion`이 20을 초과하면 해당 번호까지 동적으로 메뉴를 늘려주도록 갱신했습니다.
    - **UI 여백 확보 (Flex 수정):** 창을 너무 좁게 줄였을 때 앱 상단의 버튼들이 겹치지 않도록, `Row` 두 개로 묶여있던 헤더 영역을 `Wrap` 기반 위젯으로 교체해 여유 공간이 부족하면 자동으로 개행(줄바꿈)되도록 리팩토링했습니다.

## 3. 사후 점검 (Review)

- **Result:**
    - 지정되지 않은 엉뚱한 과목명(예: `산림필답`)이나 누락된 연도/회차 명이 들어오더라도 앱이 크래시되지 않으며, 해당 값이 유효한 선택지로서 드롭다운 창에 일시적으로 잘 보여집니다.
- **Risk Analysis:**
    - 근본 원인이었던 Assert 에러가 해소되었기 때문에 연쇄적으로 파생된 RenderFlex 오버플로우 문제도 함께 깨끗하게 사라집니다. 이후에도 AI가 제목을 잘못 해석하여 `없는 과목`을 파싱하더라도, 앱이 터지지 않고 일단 화면에 값을 띄운 뒤 관리자가 클릭해서 올바른 과목으로 변경할 수 있도록 유연성을 확보했습니다.
