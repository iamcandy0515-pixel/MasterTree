# Tree Hints Template 사용 가이드

## 📋 개요

이 템플릿은 수목의 대표힌트, 잎힌트, 수피힌트, 겨울눈힌트를 구조화된 형식으로 업데이트하기 위한 JSON 템플릿입니다.

---

## 📁 파일 구조

```
scripts/
├── data/
│   └── tree_hints_template.json          # 힌트 업데이트 템플릿
└── update_tree_hints_from_template.ts    # 업데이트 실행 스크립트
```

---

## 🎯 템플릿 형식

## 📝 힌트 작성 가이드

### 1. **대표\_힌트** (Main Image Hint)

**형식:**

```
침엽수/활엽수·상록수/낙엽수·양수/음수·교목/관목·개화 월 | 주요특징1 | 주요특징2 | 무엽기: 겨울눈 특징, 수피 특징
```

**예시:**

```
침엽수·상록수·음수·교목·개화 5월 | 잎: 끝이 오목 | 구과: 위로 섬 | 무엽기: 겨울눈 둥글고 수지, 수피 회갈색
```

---

### 2. **잎\_힌트** (Leaf Hint)

**항목:**

- 가장자리(결각): 침형/난형/타원형 등, 결각 유무
- 배열: 어긋나기/마주나기/돌려나기, 2열 배열 등
- 구조·형태: 단엽/겹잎, 형태 설명
- 잎맥: 단일맥/평행맥/그물맥
- 표면: 광택, 털, 기공조선 등
- 기타: 향, 잎자루, 특이사항

**형식:**

```
가장자리: [내용] | 배열: [내용] | 구조·형태: [내용] | 잎맥: [내용] | 표면: [내용] | 기타: [내용]
```

**예시:**

```
가장자리: 침형, 결각 없음 | 배열: 가지에 2열 배열 | 구조·형태: 단엽, 편평한 선형 | 잎맥: 단일맥 | 표면: 뒷면 기공조선 뚜렷 | 기타: 향 약함, 잎자루 없음
```

---

### 3. **수피\_힌트** (Bark Hint)

**항목:**

- 갈라짐 형태: 세로 균열/가로 균열/판상 등
- 질감·두께: 얇다/두껍다, 거칠다/매끈하다
- 피목: 뚜렷함/불분명
- 가지 색: 회갈색/적갈색/흑갈색 등
- 가시 유무: 있음/없음
- 단면(골수): 원형/별형, 수지 유무

**형식:**

```
갈라짐 형태: [내용] | 질감·두께: [내용] | 피목: [내용] | 가지 색: [내용] | 가시 유무: [내용] | 단면(골수): [내용]
```

**예시:**

```
갈라짐 형태: 얕은 세로 균열 | 질감·두께: 얇고 거칠다 | 피목: 불분명 | 가지 색: 회갈색 | 가시 유무: 없음 | 단면(골수): 원형, 수지 많음
```

---

### 4. **겨울눈\_힌트** (Bud Hint)

**항목:**

- 비늘(아린) 개수: 다수/2개/없음 등
- 털 유무: 있음/없음
- 위치: 정생/측생
- 엽흔 모양: V자형/U자형/원형 등
- 관속흔 개수: 1개/3개/다수 등

**형식:**

```
비늘(아린) 개수: [내용] | 털 유무: [내용] | 위치: [내용] | 엽흔 모양: [내용] | 관속흔 개수: [내용]
```

**예시:**

```
비늘(아린) 개수: 다수 | 털 유무: 없음 | 위치: 측생 | 엽흔 모양: 침엽이라 불분명 | 관속흔 개수: 자료없음
```

## 🚀 사용 방법

### 1단계: 템플릿 파일 편집

`scripts/data/tree_hints_template.json` 파일을 열어서 업데이트할 수목 정보를 추가합니다.

```json
[
    {
        "수목명": "분비나무",
        "tree_id": 36,
        "대표_힌트": "...",
        "잎_힌트": "...",
        "수피_힌트": "...",
        "겨울눈_힌트": "..."
    },
    {
        "수목명": "다른수목",
        "tree_id": 99,
        "대표_힌트": "...",
        "잎_힌트": "...",
        "수피_힌트": "...",
        "겨울눈_힌트": "..."
    }
]
```

### 2단계: 업데이트 스크립트 실행

```bash
cd nodejs_admin_api
npx ts-node scripts/update_tree_hints_from_template.ts
```

### 3단계: 결과 확인

스크립트 실행 후 다음 정보가 출력됩니다:

- 처리된 수목 수
- 업데이트된 힌트 수
- 새로 생성된 힌트 수
- 오류 수

---

## 📊 Tree ID 확인 방법

Tree ID를 모르는 경우:

```bash
npx ts-node scripts/check_db_status.ts
```

또는 Supabase Dashboard에서 확인:

```sql
SELECT id, name_kr FROM trees ORDER BY name_kr;
```

---

## ⚠️ 주의사항

1. **tree_id 필수**: 각 수목의 정확한 tree_id를 입력해야 합니다.
2. **선택적 필드**: 대표*힌트, 잎*힌트, 수피*힌트, 겨울눈*힌트는 모두 선택사항입니다. 필요한 것만 입력하세요.
3. **구분자**: 각 항목은 `|` (파이프)로 구분합니다.
4. **일관성**: 동일한 형식을 유지하여 데이터 일관성을 확보하세요.

## 📁 관련 파일

- `tree_hints_template.json` - 힌트 데이터 템플릿
