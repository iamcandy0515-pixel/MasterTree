# 기출문제 추출 화면 UI 및 기능 개선 작업 내역

## 1. 작업 목적 (Plan)

- 기출문제 연동(추출) 화면에서 사용자가 선택한 과목, 년도, 회차 조건을 시각적으로 표시하고, 잘못된 파일 추출을 방지하기 위해 필터 검증 기능을 추가합니다.
- 계산 문제 추출 시 수식(LaTeX 등)이나 마크다운 문법이 아닌 순수 텍스트 형식으로 파싱되도록 AI 프롬프트를 개선합니다.
- 불필요한 추출 조건 콤보박스(과목, 년도, 회차)를 삭제하여 UI를 간소화합니다.

## 2. 작업 내역 (Execute)

- **UI 개선:**
    - `QuizManagementScreen`에서 `QuizExtractionStep2Screen`으로 과목, 년도, 회차 값 전달 로직 추가.
    - `QuizExtractionStep2Screen`의 '구글 드라이브 연동' 상단에 전달받은 조건을 '검색필터' 형태로 시각화.
    - `PdfExtractionModule` 에서 추출 조건 영역에 있던 '과목', '년도', '회차' 드롭다운을 삭제하고, '문제번호' 레이블 텍스트 새로 추가.
- **기능 및 검증 추가:**
    - `QuizExtractionStep2ViewModel` 에 필터 상태(`initialSubject`, `initialYear`, `initialRound`) 저장용 변수 및 setter 추가.
    - `extractQuiz` 실행 시 검색 필터 조건과 문서 식별자(파일명 파싱 등)가 불일치할 경우, "해당 파일은 조건에 맞지 않습니다. 다른 파일을 입력하세요 (...)" 예외를 발생시키도록 로직 추가.
- **Node.js 프롬프트 수정:**
    - `nodejs_admin_api/src/modules/quiz/quiz.service.ts`의 `extractQuizFromPdfBuffer` 프롬프트를 수정하여 계산 문제의 '정답 및 해설' 블록 생성 시 완벽한 평문(Plain Text) 및 한글 텍스트 전용 기호만 사용하도록 명확한 금지 규칙(`LaTeX` 및 마크다운 지양)을 추가 적용.

## 3. 향후 문제점 및 리스크 분석 (Risk Analysis)

- 파일명 기반 필터 검증 로직이기 때문에, 드라이브에 업로드된 파일의 이름 작성 규칙(과목*년도*회차 형식 등)이 올바르지 않으면 정상 파일도 오류를 발생시킬 수 있습니다.
- AI 프롬프트를 강하게 제한하였으나 언어 모델의 특성상 낮은 확률로 특수문자가 섞일 여지는 존재하므로 추출 결과를 검수하는 단계가 필수적입니다.
