# 기출문제 연동 - PDF 없는 문제 번호 추출 환각(Hallucination) 방지 (quiz-extraction-hallucination-prevention)

## 1. 목적 및 원인 파악 (Plan)

- **증상:** "산림필답\_2013_1" 등 총 문항수가 15번까지밖에 없는 파일인데도 16번 문제 추출 클릭 시 기출문제가 뽑혀나옴.
- **원인:** LLM (Gemini) 특유의 **환각(Hallucination)** 현상. AI가 문맥상 이전 문제 내용들을 바탕으로 16번 문제가 있을 법한 내용(산림/조경 분야)을 스스로 창작해서 반환한 결과임.
- **해결 방안:** 백엔드(Node.js API) 프롬프트에 초강력 제약 규칙을 부여하고, 프론트엔드(Flutter app)에서는 이 조치로 인해 아무것도 반환되지 않았을 경우 (빈 배열 `[]` 반환 시) 사용자 친화적인 에러 메세지를 띄워 화면 렌더링을 차단한다.

## 2. 작업 내용 (Execute)

- **Node.js (API):** `quiz.service.ts`의 `extractQuizFromPdfBuffer` 내부 프롬프트 최상단에 환각 방지 문구 추가 완료.
    ```typescript
    **🚨[가장 중요한 규칙 - 환각(Hallucination) 방지]🚨**
    문서 내에 정확히 해당 번호가 존재하지 않는다면 어떤 경우라도 문제를 창작하지 말고 완벽하게 파싱을 중단하며 **빈 배열 []** 만을 반환하세요.
    ```
- **Flutter (Client):** `quiz_extraction_step2_viewmodel.dart` 및 화면 단에서 `parsedBlocks`가 빈 값인 경우 `throw '선택하신 문제 번호가 해당 파일(PDF)에 존재하지 않습니다.'` 처리를 하여 `SnackBar` (토스트 메시지)로 사용자에게 즉각 보여지도록 개선 완료.

## 3. 사후 점검 (Review)

- **Result (완료된 결과):** 존재하지 않는 문제 번호로 PDF 추출을 시도할 경우, AI가 임의의 문제를 지어내지 않고 빈 값을 던져주어 프론트엔드에 즉시 "해당 파일(PDF)에 존재하지 않습니다" 라고 뜨게 됨.
- **Risk Analysis (향후 문제점 및 리스크 분석):** PDF 스캔 품질이 너무 안 좋거나 번호의 레이아웃이 독특하게 깨진 경우, 실제로 존재하는 문제임에도 AI가 "번호를 못 찾았다"고 판단하여 추출하지 않고 튕겨버릴 가능성 (False Negative) 이 존재. 만일 이 빈도가 잦아질 경우 빈 배열 조건보다는 프롬프트 유연성을 어느 정도 줘서 "N번 문제가 없으면 그 다음 내용을 강제로 파싱하라"는 등의 우회 대안책 또는 OCR 화질 개선이 고려될 수 있음.
