# [작업계획서] 사용자 대시보드 구조 개선 및 모바일 최적화 (V2.1)

## 1. 프로젝트 목적 및 범위
`DEVELOPMENT_RULES.md` 가이드라인에 따라 사용자 앱(`flutter_user_app`)의 핵심인 대시보드를 리팩토링하여 기능적 완성도를 높이고, 모바일 환경에서의 성능(로드 부하/경량화)을 최적화합니다.

## 2. 주요 작업 내역 (UI/UX 및 기능)

### 2.1 대시보드 레이아웃 및 디자인 (Standardization)
- **헤더(AppBar) 개편**:
    - 뒤로가기 아이콘(`<`) 제거 및 좌측에 '나가기(로그아웃)' 아이콘 배치
    - 사용자 앱 타이틀 폰트 크기를 관리자 앱 가이드라인(16~18pt)에 맞춰 하향 조정하여 세련된 디자인 유지
- **탭 시스템 도입**: '학습 모듈' 헤더 제거 후 2분할 탭 구성
    - **탭 1: 수목관리** (수목도감 일람, 수목/퀴즈, 비교 수목 일람)
    - **탭 2: 기출문제** (기출/학습)
- **공간 최적화**: 카드 간 세로 마진(Vertical Spacing)을 정밀 조정하여 한 화면 내 시인성 극대화 (Compact Layout)

### 2.2 화면별 세부 기능 고도화
- **수목도감 일람**: 검색 필터와 2단계 콤보 필터(침엽·활엽 / 상록·낙엽) 간의 유기적 연동 로직 강화
- **수목 / 퀴즈**: 화면 제목을 '수목 식별 학습'에서 **'수목/퀴즈'**로 변경
- **비교 수목 일람**: 페이지네이션 UI에 **`맨 처음(first_page)`**, **`맨 마지막(last_page)`** 이동 버튼 추가
- **기출 / 학습**:
    - 리스트 상단에 선택된 필터 기준 **'총 문제 건수'** 실시간 표시
    - 문제 번호 형식을 `01`에서 **`1번. `** 형식으로 일괄 변경 (상세 화면 연동)

## 3. 모바일 로드부하 저감 및 리팩토링 전략 (Compliance)

### 🚀 성능 최적화 (Lightweight Optimization)
- **Sliver 기반 지연 로딩**: `CustomScrollView`와 `SliverList`를 사용하여 화면 노출 영역만 메모리에 상주시켜 CPU/GPU 부하 저감
- **Partial Re-build**: `ValueListenableBuilder`와 `ChangeNotifier`를 활용하여 탭 전환 시 전체 화면이 아닌 특정 위젯만 리렌더링
- **이미지 메모리 관리**: `CachedNetworkImage` 활용 시 `memCacheWidth`를 제한하여 텍스트 데이터 중심의 가벼운 앱 환경 조성

### 📦 코드 구조 개선 (200-Line Rule)
- **위젯 분리**: 대시보드 탭 내용을 `DashboardTreeTab`, `DashboardExamTab` 등 별도 파일로 분리하여 모듈화
- **로직 관심사 분리**: 모든 검색 및 상태 변경 로직은 `Controller` 레이어에서 관리 (Screen 파일 내 로직 0% 지향)
